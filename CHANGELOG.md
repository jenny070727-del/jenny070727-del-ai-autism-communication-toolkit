# Changelog

## 2026-07-14｜v0.2.3

### Added

- 页面顶部新增“产品说明与安全边界”入口。
- 说明中明确：
  - 本工具不是诊断、治疗或标准评估；
  - 实体图片卡只是 AAC 的一种形式，成熟 AAC 也可以是沟通板、文字、手势、平板 App、语音生成设备、眼动或开关扫描；
  - “当前沟通方式”不是能力评估，只是影响输出形式的产品参数。

### Changed

- “当前沟通方式”改为全局设置，影响三个模块输出。
- 本地兜底输出现在会根据沟通方式调整：
  - 视觉流程的步骤数量和文字量；
  - 表达卡数量、备用表达和卡片类型；
  - 沟通伙伴脚本中的等待、指卡、手势选择或备用表达提示。
- API 请求会把 `communication_mode` 传给 Cloudflare Function，部署后 AI 生成也会参考该参数。

## 2026-07-14｜v0.2.2

### Changed

- 简化三个模块的主交互：统一为“输入真实场景/成人原话 → 生成结果”。
- 模块 1 删除预设流程按钮、支持目标下拉和场景下拉。
- 模块 1 的“孩子当前情况”改为可选高级设置；后续再让它更明显影响输出。
- 模块 2 删除预设按钮，直接根据场景描述生成核心卡、场景卡和功能表达卡。
- 模块 3 删除预设按钮，直接根据成人原话生成低压力短句和示范脚本。
- 生成后自动滚动到输出区域，减少“点了但不知道有没有反应”的问题。

## 2026-07-14｜v0.2.1

### Changed

- 重新整理三模块边界，减少模块 1 和模块 2 的重复。
- 模块 1 从“视觉支持综合包”调整为“视觉流程支持”：
  - 视觉日程表；
  - First-Then；
  - 社交叙事；
  - 成人展示建议。
- 模块 2 从“简单 AAC 卡片生成器”调整为“表达卡系统”：
  - 固定核心卡；
  - 当前场景卡；
  - 功能表达卡。
- 模块 3 从“语言简化器”扩展为“沟通伙伴支持”：
  - 成人低压力短句；
  - Aided Language Modeling 示范脚本；
  - 等待与确认提醒。

### Added

- `/api/generate` 支持三种模式：
  - `visual`
  - `aac`
  - `partner`
- README、PROGRESS、ROADMAP、PRD 同步更新为 v0.2.1 结构。

### Safety

- 继续保持非医疗定位：
  - 不诊断；
  - 不治疗；
  - 不解释行为原因；
  - 不推断隐藏意图；
  - 不自动替儿童回复。
