# 夏令营观察表｜自闭症沟通支持项目 v0.1

更新日期：2026-06-24  
用途：用于夏令营、绿房子公益项目或相关机构场景中的初步观察  
项目方向：AI 自闭症沟通支持工具包  
观察重点：视觉支持需求、成人语言复杂度、沟通误解、可复刻成熟方法、AI 可辅助环节

## 0. 使用原则

这份观察表不是诊断表，也不是评估儿童能力的表。它的目的不是判断某个孩子“好不好”“配不配合”，而是观察沟通场景中到底哪里容易卡住，以及成熟沟通支持方法可以如何帮助。

使用时请注意：

- 不记录儿童姓名、诊断、照片、录音、视频或任何可识别信息。
- 如果需要记录真实案例，必须先确认机构、家长和专业老师允许。
- 不私下测试儿童，不诱导儿童做额外任务。
- 尽量观察“场景”和“沟通过程”，不要给儿童贴标签。
- 所有判断都只是初步观察，需要康复老师或专业人员确认。
- 如果儿童表现出明显不适，应立刻停止相关互动，把安全放在第一位。

## 5. AI 可能介入的位置

记录时不要急着说“这里一定要 AI”。先判断这个问题是否可以由成熟方法解决，再看 AI 是否能降低使用门槛。

| 沟通断点 | 成熟方法 | AI 可以怎么辅助 | 优先级 |
|---|---|---|---|
| 孩子不知道接下来会发生什么 | 视觉日程表 | 自动生成当天流程图 | 高 / 中 / 低 |
| 任务太复杂 | 步骤图 | 把任务拆成 3–5 步 | 高 / 中 / 低 |
| 成人语言太复杂 | 沟通伙伴训练 | 把复杂话改写成短句 | 高 / 中 / 低 |
| 孩子难以表达拒绝或休息 | 图片沟通卡 / AAC | 生成“不要/停止/休息”卡片 | 高 / 中 / 低 |
| 场景新、孩子焦虑 | 社交故事 | 生成个性化社交故事 | 高 / 中 / 低 |
| 沟通失败后没人复盘 | 事件记录 | 帮助结构化记录与复盘 | 高 / 中 / 低 |

## 6. 每日复盘问题

每天结束后，可以回答这几个问题。

1. 今天最常见的沟通困难是什么？
2. 这些困难更像是儿童表达困难，还是成人理解/表达方式的问题，还是环境问题？
3. 哪些场景最需要提前视觉化？
4. 哪些成人表达如果改成短句，可能会更好？
5. 儿童有没有表达“不要”“停止”“休息”“没听懂”“你理解错了”的机会？
6. 今天有没有看到已有成熟方法已经很有效？
7. 今天有没有看到成熟方法有效但制作/使用成本太高？
8. 如果做 AI 工具，最应该优先解决哪个小问题？
9. 这个功能是否安全？是否可能误导成人？
10. 明天最想继续观察什么？

## 7. 向康复老师 / 绿房子负责人请教的问题

这些问题适合在合适时间请教，不一定一次问完。

### 关于儿童真实需求

- 在你们看来，孩子们最常见的沟通困难是什么？
- 哪些场景最容易出现误解或情绪崩溃？
- 孩子们通常如何表达“不要”“停止”“我累了”“我没听懂”？
- 有没有一些孩子其实有表达意愿，但成人很难理解？

### 关于现有方法

- 你们现在最常用的沟通支持方法有哪些？
- 视觉日程表、图片卡、社交故事、沟通板这些方法是否常用？
- 哪些方法有效，但家长很难长期坚持？
- 哪些材料制作起来最耗时间？

### 关于家长和志愿者

- 家长最容易遇到的困难是什么？
- 志愿者和孩子互动时最需要注意什么？
- 普通人最容易犯的沟通错误是什么？
- 如果有一个 AI 工具，最希望它帮家长/老师/志愿者做什么？

### 关于验证方式

- 如果我们做一个很初步的工具，怎样验证比较安全？
- 是让康复老师先看，还是让家长先用，还是在活动后复盘？
- 哪些数据可以记录，哪些不应该记录？
- 什么样的反馈能说明这个工具有帮助？

## 9. 判断一个功能是否值得做的标准

一个功能如果满足以下条件，就比较适合进入第一版原型：

- 它对应已有成熟方法，而不是凭空发明。
- 它不会让 AI 直接判断孩子真实意图。
- 它能帮助家长、老师或志愿者更容易使用成熟方法。
- 它能在真实场景中产生可观察的反馈。
- 它不会增加儿童压力。
- 它能保留儿童表达拒绝、停止、休息和纠正误解的权利。

第一版优先做“低风险、高频、容易验证”的功能。
