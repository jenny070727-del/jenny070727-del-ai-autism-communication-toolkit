# V0.2.2 PRD

PRD = Product Requirements Document。

## 项目目标

做一个安全、低门槛的 AI 沟通支持工具包，用 AI 复刻和降低已有成熟方法的制作成本，而不是发明新的自闭症干预方案。

## 核心原则

- 安全优先；
- 先复刻成熟方法，再考虑创新；
- 不做诊断、治疗、行为解释或隐藏意图推断；
- 输出只是草稿，需要成人和专业人员确认；
- 目标不是让孩子“更听话”，而是降低双方沟通成本。

## 用户

第一阶段假设用户是：

- 助教；
- 营地老师；
- 家长；
- 康复老师或专业负责人。

## 三个模块

三个模块的交互原则：

```text
输入真实场景 / 成人原话 → 生成结果
```

不把预设按钮作为主入口；不保留暂时不会明显影响输出的下拉选项。

### 模块 1：视觉流程支持

用户输入一个即将发生的场景。

可选输入：

- 孩子当前情况。

系统输出：

- 视觉日程表；
- First-Then / 先后图；
- 社交叙事；
- 成人展示建议；
- 建议搭配模块 2 的表达卡名称。

对应成熟方法：

- Visual Schedule；
- First-Then；
- Social Narrative。

### 模块 2：表达卡系统

用户输入一个真实场景。

系统输出：

- 固定核心卡；
- 当前场景卡；
- 功能表达卡；
- 使用提醒。

对应成熟方法：

- AAC / 图片沟通；
- Core Vocabulary；
- Fringe Vocabulary；
- Functional Communication。

### 模块 3：沟通伙伴支持

用户输入成人原话或沟通场景。

系统输出：

- 低压力短句；
- Aided Language Modeling 示范脚本；
- 等待与确认提醒。

对应成熟方法：

- Communication Partner Training；
- Aided Language Modeling；
- Wait + Confirm。

## 非目标

- 不诊断；
- 不治疗；
- 不声称改善症状；
- 不解释行为原因；
- 不推断隐藏意图；
- 不根据哭闹、沉默、眼神等判断真实想法；
- 不收集可识别儿童数据；
- 不替儿童自动回复。

## 成功标准

- 用户能在 2 分钟内理解三个模块的区别；
- 每个模块都能根据用户输入生成不同结果；
- 本地预览时有兜底输出；
- 部署后可以调用 `/api/generate`；
- 输出符合安全边界；
- 项目材料能解释它参考了哪些成熟方法。
