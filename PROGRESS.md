# Progress

Last updated: 2026-07-09

## Current status

The project currently has a `v0.1` clickable concept prototype.

The prototype demonstrates the main idea: using AI to lower the barrier to established autism communication support methods, rather than inventing a risky new intervention from scratch.

## Completed

### Project direction

- Defined the project as an AI-assisted communication support toolkit.
- Clarified that the first stage should reproduce mature methods before attempting innovation.
- Defined the core principle: safety first, mature methods second, AI as a helper.
- Positioned the long-term goal as supporting two-way communication between autistic people and communication partners.

### Research framing

- Identified mature methods worth referencing:
  - visual schedules;
  - step-by-step visual routines;
  - social stories;
  - AAC-inspired picture communication;
  - communication partner training;
  - structured communication event review.
- Clarified what AI should not do:
  - diagnose;
  - infer hidden intentions;
  - replace professionals;
  - automatically speak for the child;
  - treat “normalization” as the goal.

### Prototype

- Built `v0.1` static HTML prototype.
- Included three modules:
  - Visual Support Generator;
  - Communication Partner Coach;
  - Communication Event Review Assistant.
- Added visible safety statements.
- Designed the prototype for discussion with mentors, professors, and organization staff.

### Project documents

- Drafted a Chinese one-page project brief.
- Drafted a Chinese summer camp observation form.
- Created this GitHub-ready project folder.
- Added README, roadmap, progress file, and research notes.

## Not completed yet

- The prototype does not connect to a real AI model.
- Generated outputs are still preset examples.
- There is no backend or database.
- No real user testing has been conducted.
- No therapist, educator, parent, or caregiver has reviewed the tool yet.
- No data has been collected.
- The tool has not been validated in any autism support setting.

## Next steps

### Short term

1. Upload the project to GitHub.
2. Deploy the static prototype through Cloudflare Pages or GitHub Pages.
3. Ask mentors or relevant professionals to review the wording and safety boundaries.
4. Decide whether `v0.2` should focus first on visual support generation or communication partner coaching.

### Before real-world observation

1. Confirm what can and cannot be observed.
2. Avoid collecting identifiable child information.
3. Use the observation form to focus on communication scenes rather than judging individual children.
4. Ask professionals how a prototype can be safely reviewed.

### Medium term

1. Add real AI generation to one module.
2. Start with the lowest-risk module: Visual Support Generator.
3. Add exportable visual schedule output.
4. Add a review step for adults or professionals before any material is used.
5. Collect anonymous feedback from adults, not private child data.

## Open questions

- What safety language should be reviewed by professionals?

- What counts as useful feedback without making clinical claims?
- How can the tool preserve children’s autonomy, especially refusal and correction?

## Current best project summary

This project explores whether AI can make established autism communication support methods easier for parents, teachers, and volunteers to use. The first stage focuses on visual supports, communication partner coaching, and structured case review. It is not a diagnostic or therapeutic tool. Its purpose is to support safer, clearer, more respectful communication and to prepare for future research through careful observation and feedback.
