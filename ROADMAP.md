# Roadmap

## v0.2.3：产品说明与沟通方式设置

目标：把敏感边界说清楚，并让“当前沟通方式”真正影响输出，而不是只作为展示选项。

已完成：

- 页面顶部新增“产品说明与安全边界”；
- 明确本工具不是诊断、治疗或标准评估；
- 明确实体卡片只是 AAC 的一种形式；
- “当前沟通方式”改成全局设置；
- 当前沟通方式会影响视觉流程、表达卡系统和沟通伙伴支持；
- API prompt 同步接收 `communication_mode`。

## v0.2.2：简化交互版

目标：三个模块都变成“输入真实场景/成人原话 → 生成材料”，减少预设按钮和暂时无效的下拉选项。

已完成：

- 模块 1 删除预设流程按钮、支持目标下拉和场景下拉；
- 模块 2 删除预设按钮；
- 模块 3 删除预设按钮；
- 模块 1 的孩子当前情况折叠为高级设置；
- 生成后自动滚动到输出区。

## v0.2.1：清晰三模块结构

目标：把成熟沟通支持方法融入现有三块，而不是继续增加混乱的新模块。

### 模块 1：视觉流程支持

- 视觉日程表
- First-Then / 先后图
- 社交叙事
- 成人展示建议

### 模块 2：表达卡系统

- 核心词卡
- 当前场景卡
- 功能表达卡
- 使用提醒

### 模块 3：沟通伙伴支持

- 成人语言简化
- Aided Language Modeling 示范脚本
- 等待与确认提醒

## v0.3：真实场景测试

- 用夏令营观察到的真实场景测试输出是否有用。
- 记录哪些输出需要专业人员修改。
- 区分哪些场景适合视觉支持，哪些场景更需要表达卡或成人语言简化。

## v0.4：更像工具，而不是页面 Demo

- 支持保存多个场景。
- 支持复制单个模块输出。
- 支持打印或导出 PDF。
- 支持更清楚的“核心卡 / 场景卡 / 功能卡”布局。

## 暂不做

- 诊断；
- 治疗建议；
- 情绪识别；
- 行为原因解释；
- 事件复盘 AI；
- 自动替儿童回复。
